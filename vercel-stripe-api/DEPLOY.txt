MyAISNS 用 Stripe API（Vercel 専用サブフォルダ）
==============================================

■ あなたがやること（5 分程度）

1) Vercel に「新規プロジェクト」を作る。
   - GitHub などと連携する場合: リポジトリを選び、
     Root Directory を「vercel-stripe-api」に設定する。
   - またはこのフォルダだけ zip して Vercel にドラッグでも可。

2) プロジェクトの Environment Variables に追加:
   STRIPE_SECRET_KEY = sk_test_...（Stripe ダッシュボード → 開発者 → API キー）

3) Deploy。完了後の URL をメモする（例: https://xxxx.vercel.app）

4) MyAISNS プロジェクト直下に .env を作り（なければ）、次を記入:
   EXPO_PUBLIC_USE_STRIPE_CHECKOUT=1
   EXPO_PUBLIC_STRIPE_CHECKOUT_API_URL=https://xxxx.vercel.app/api/stripe-checkout
   EXPO_PUBLIC_STRIPE_VERIFY_API_URL=https://xxxx.vercel.app/api/stripe-verify

5) Expo を再起動（npx expo start）。ショップから Checkout を試す。

■ 備考

- 本番では sk_live_... と本番用 price_xxx を使う。
- コイン付与をより安全にするなら Stripe Webhook を別途検討。

==============================================
■ 本番（ライブ）に切り替える手順
==============================================

A) Stripe ダッシュボード
  1. 画面上部で「本番データ」モードに切り替え（テストから本番へ）。
  2. 設定 → ビジネス設定で「アカウントを有効化」／本人確認・事業情報を完了する。
     （未完了だと本番の課金に制限がかかることがあります）
  3. 商品カタログで、テストと同じ構成の「一回限り」価格（JPY）を本番側でも作成する。
     → 本番用の price_xxxxxxxx をメモする（テストの ID とは別）。

B) アプリのコード（MyAISNS）
  1. src/services/stripeService.ts の COIN_PACKS[].priceId を
     すべて「本番モードで作った price_xxx」に差し替える。
  2. 金額表示 priceJpy も Stripe の本番価格と一致させる。

C) Vercel（my-aisns など）
  1. Project → Settings → Environment Variables
  2. STRIPE_SECRET_KEY を sk_live_.... に変更（本番用シークレットキー）。
     Production（および Preview にも同じキーを使うか方針を決める）
  3. Save 後、Deployments → 最新を Redeploy（環境変数反映のため）。

D) アプリのビルド
  - ローカル .env は本番用に書き換えない運用でもよい（EAS の場合）。
  - EAS Build なら eas secret で EXPO_PUBLIC_* を本番用 URL のまま設定。
  - 本番 APK/IPA には「テスト用 sk」を絶対に埋め込まない（クライアントには入れない）。

E) 動作確認
  - 本番では実カードで課金されるため、最初は最小額の商品で 1 件だけ試すか、
    Stripe の「テストモード」と「本番モード」を混同しないこと。

■ 公式サイト（Stripe の「ビジネスのウェブサイト」用）

  public/index.html がランディングページです。デプロイ後は
  https://（プロジェクト）.vercel.app を Stripe に入力できます。
  public/tokushoho.html が「特定商取引法に基づく表記」です。
  Stripe 審査では住所・電話・代表者名を登録情報と一致させる必要があるため、
  tokushoho.html の【】プレースホルダを必ず実データに置き換えてから push する。

F) さらに安全にする（推奨・別タスク）
  - checkout.session.completed を受ける Webhook を Vercel に追加し、
    サーバーだけが「支払い済み」を確定する。
  - コイン残高を Supabase に持つなら、Webhook 内でユーザにコインを加算する設計が定石。
